Project Overview

This is a demonstration of a modern containerized web application running on AWS ECS. The project showcases best practices for cloud-native application deployment, infrastructure as code, and continuous deployment pipelines.

  • Container Platform: AWS ECS with Fargate launch type
  • Load Balancer: Application Load Balancer (ALB)
  • Networking: VPC with public subnets (free tier optimized)
  • Security: Security groups with least privilege access

üí∞ Free Tier Optimizations

This project is specifically configured to run within AWS Free Tier limits:

  • Region: us-east-1 for best free tier availability
  • Compute: Minimal Fargate resources (0.25 vCPU, 0.5 GB RAM)
  • Networking: No NAT Gateway to avoid charges
  • Logging: Minimal log retention (1 day)
  • Monitoring: Container Insights disabled
  • Storage: No persistent storage to avoid EBS charges

üîÑ Deployment Process

The application uses a fully automated CI/CD pipeline:

1. Code Push ‚Üí GitHub Repository 2. GitHub Actions Triggered 3. Docker Image Built & Pushed to ECR 4. ECS Service Updated with New Image 5. Health Checks Verify Deployment 6. Traffic Routed to New Tasks

üîß Getting Started
